DAG 布局算法设计文档

Status: draft（待确认） · Created: 2026-06-29

DAG 小地图把每个节点放到网格的 (列, 行)。本文定义三件事：节点放第几列、
第几行。规则要简单——不绕弯。

一、一个节点的位置 = (x, y)

  • x（横向位置）= lane列 + tier缩进
  • y（纵向位置）= depth行号

三个量：lane、tier、depth。下面逐个定义，每个都尽量简单。

二、lane（这个节点属于第几条分支）

规则：有几条分支，就排几列。按分支出现的先后顺序编号，从 0 开始，不跳号。

  • 第 1 条分支 → lane 0
  • 第 2 条分支 → lane 1
  • 第 3 条分支 → lane 2
  • ……

什么是"一条分支"：从一句用户消息开始，顺着对话往下（user → AI回复 → 下一句

user → AI回复 …），这一整条就是一条分支。中途如果你 retry / 改写某句话，就在

那个点岔出一条新分支，新分支拿下一个编号。

例：只发了一句"你好"（没 retry）

→ 只有 1 条分支 → 全部 lane 0。不存在第 2 列。

例：发了"你好"，又 retry 改成"在吗"

→ 2 条分支 → 原来那条 lane 0，retry 出来的那条 lane 1。

不要做的事（之前代码绕的弯，全删）：

  • ❌ 不要专门判断"哪条是主干"
  • ❌ 不要"归零"之类的特殊处理
  • ❌ 不要让没有分叉的对话也占用第 2 列

就是：数分支，按顺序发列号。

三、tier（节点在自己这列里往右缩进几格）

规则：按角色固定缩进，表示"谁套在谁里面"。

  • ROOT（会话根）：缩进 0
  • user（用户消息）：缩进 1
  • AI 回复：缩进 2
  • AI 调用的工具：缩进 3
  • 工具内部再调的东西：再 +1

这就是那个"阶梯往右"的效果——这是对的，本来就该这样缩进，不是 bug。

下一轮 user 回到缩进 1（不是一直往右涨）。

四、depth（节点在第几行，从上往下）

规则：按聊天/执行的先后顺序，从上往下一行一行排。

  • 越早发生的越靠上。
  • 岔出来的分支，和它岔出的那个位置对齐到同一行起步，然后各自往下长。

五、画出来长什么样

只发一句"你好"（1 条分支，全 lane 0）：

列:   0    1    2
行0  ◇ROOT
行1       ○你好          ← lane 0, 缩进 1
行2            △AI回复    ← lane 0, 缩进 2

发"你好" + retry 改成"在吗"（2 条分支）：

列:   0    1    2         3    4
行0  ◇ROOT
行1       ○你好  ┈┈┈┈┈┈┈ ○在吗       ← 第2条分支 lane 1
行2            △回复1          △回复2

（虚线表示这两条是从同一个点岔开的平行分支）

六、当前 bug

只发了一句"你好"，本该全 lane 0，但代码给 user/AI回复 发了 lane 1（跳过了

0），导致它们整体往右偏一大块，ROOT 孤零零在左边。

根因：代码判断"分支怎么分列"时用错了一条线（用了"对话前驱"那条，而第一句话

没有前驱 → 接不上 → 被误判成新分支），还套了一层"主干判断 / 归零"的绕弯逻辑。

**修法：照第二节的规则重写——数分支、按出现顺序发列号 0,1,2，删掉主干判断和

归零那些东西。**

七、怎么验证（不用截图网页）

python tools/dag_dump.py [session_id]

直接在命令行打印每个节点的 lane / tier / depth + ASCII 网格图。改完代码跑一下

就能看对不对。

要验的三种情况：

1. 只发一句话 → 全 lane 0（当前 bug，修完应为 0）

2. retry 出第二条分支 → 第二条 lane 1

3. 调用工具 → 工具在 AI 回复下面缩进一格（tier+1），不另开 lane
