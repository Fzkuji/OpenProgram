## 一、一个节点的位置 = (列, 行)

- **列（横向）= lane起始列 + tier缩进**
- **行（纵向）= depth**

三个量：**lane**（属于第几条分支）、**tier**（在分支内缩进几格）、**depth**
（第几行）。下面逐个定义。

## 二、lane —— 这个节点属于第几条分支

**规则：有几条分支，就排几列。按分支出现的先后顺序编号，从 0 开始，不跳号。**

- 第 1 条分支 → lane 0
- 第 2 条分支 → lane 1
- 第 3 条分支 → lane 2 ……

**一条分支** = 一条对话链：从一句 user 消息开始，顺着对话往下（user → AI回复 →
下一句 user → AI回复 …）连成的整条。中途 retry / 改写某句话，就在那个点**岔出
一条新分支**，新分支拿下一个编号。

| 场景 | 分支数 | lane |
|---|---|---|
| 只发一句"你好"（没 retry） | 1 | 全 lane 0，不存在第 2 列 |
| "你好" + retry 改成"在吗" | 2 | 原 lane 0，retry 出来的 lane 1 |

**绝不要做**（之前代码绕的弯，全删）：
- ❌ 专门判断"哪条是主干"
- ❌ "归零"之类特殊处理
- ❌ 让没有分叉的对话占用第 2 列

就是：**数分支，按出现顺序发列号。**

## 三、tier —— 在分支内往右缩进几格

**规则：按角色固定缩进，表示"谁套在谁里面"（沿 `caller` 子调用边）。**

| 节点 | tier |
|---|---|
| ROOT | 0 |
| user | 1 |
| AI 回复（llm） | 2 |
| AI 调用的工具（code） | 3 |
| 工具内部再调的（子调用） | 上一级 +1 |

"阶梯往右"的缩进效果是**对的**，不是 bug。下一轮 user 回到 tier 1（不是一直
往右涨）——因为所有顶层 user 都直接挂在 ROOT 下，平级。

## 四、depth —— 第几行，从上往下

**规则：按聊天/执行的先后顺序（`predecessor` 对话链 + seq），从上往下排。**

- 越早发生的越靠上。
- 岔出来的分支，和它岔出的位置**对齐到同一行**起步，然后各自往下长。

## 五、三条全局排版规则

**① 正方形网格**：横向格距 = 纵向格距（`COL_W == ROW_H`）。整张图是标准棋盘格，
每列列距、每排行距完全一致。子节点在父节点的**严格右下角**——往右一格 + 往下
一格（45° 对角线，距离 √2×格距）。

**② 严格网格对齐 + 横竖都紧凑**：每个节点落在网格交点上，同一行 y 完全对齐、
同一列 x 完全对齐，不允许半格偏移。**列空了右边左移补上，行空了下面上移补上**
——收起一个节点，它占的那一行腾空，下面所有行整体上移；它占的列腾空，右边的
列左移。空行、空列都不保留。

**③ 分支按实际占用列紧贴排，不重叠**：一条分支占用的列 = 从它的起始列到它子树
最深那一格。**下一条分支从上一条实际占用的最右列 +1 开始**，整条在前一条右边，
互不重叠。

> 规则 ②③ 联动 = "收起左移"：前一条分支的子调用收起 → 它占用的列变少 → 后面
> 分支起始列跟着左移。例：lane0 展开占列 0~3，lane1 从列4 起；工具收起后 lane0
> 只占 0~2，lane1 左移到列3，同时下面的行也上移。

## 六、画出来长什么样（文字版；图示见 spec.html）

**只发一句"你好"（1 条分支，全 lane 0）：**
```
列:   0    1    2
行0  ◇ROOT
行1       ○你好          ← lane0, tier1
行2            △AI回复    ← lane0, tier2
```

**"你好" + retry 改成"在吗"（2 条分支）：**
```
列:   0    1    2    3    4
行0  ◇ROOT
行1       ○你好 ┈┈┈ ○在吗          ← lane1 从列3 起（lane0 占列0~2）
行2            △回复1    △回复1'
```
lane0 占列 0~2 → lane1 从列 3 起，紧贴不重叠。虚线 = 从同一点岔开的平行分支。

## 七、当前 bug（要修的）

只发一句"你好"，本该全 lane 0，但代码给 user/AI回复 发了 **lane 1**（跳过 0），
导致整体往右偏一大块，ROOT 孤零零在左边。

根因：判断"分支怎么分列"时用错了边——用了 `predecessor`（对话前驱），而第一句
话没有前驱 → 接不上 → 被误判成新分支；还套了一层"主干判断 / 归零"的绕弯逻辑。

**修法：照第二节重写——数分支、按出现顺序发列号 0,1,2，删掉主干判断和归零。**

## 九、验证（命令行，不用截图网页）

```
python tools/dag_dump.py [session_id]
```
打印每个节点的 lane / tier / depth + ASCII 网格图。改完代码跑一下对照。

要验的场景（对应 spec.html 7 张图）：
1. 单轮："你好" → 全 lane 0
2. 多轮：所有 user 挂 ROOT、tier1 同列；全 lane 0
3. retry：第 2 条分支 lane1，从 lane0 最右列 +1 起
4. 工具调用：工具 tier3，在 AI 下缩进，不另开 lane
5. 手动调函数：函数挂 ROOT（tier1），内部子调用逐层缩进
6. 综合：lane 不重叠、tier 缩进、depth 按时间
7. 收起：横向 lane 左移、纵向行上移，空行空列不留
